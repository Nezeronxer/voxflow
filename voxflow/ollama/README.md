# VoxFlow + Ollama (локальный Qwen3-движок)

Полностью офлайн альтернатива облачному Gemini-слою: локальная модель **Qwen3** через
[Ollama](https://ollama.com) чистит сырую речь и форматирует текст под активное приложение
(профили FORMAL / WORK / CASUAL / VERBATIM / DOC). Аудио и текст не покидают устройство.

## Установка

1. Поставь Ollama: https://ollama.com/download (Windows). После установки сервер слушает
   `http://localhost:11434`.
2. Скачай модель:
   ```bash
   ollama pull qwen3:4b
   ```
   `qwen3:4b` хватает для скорости на CPU/слабой GPU. Точнее — `qwen3:8b` (медленнее).
   Идеальный вариант без `<think>` — `qwen3:4b-instruct` (если доступен в твоём Ollama).

3. (Опционально) Собери «запечённую» модель `voiceflow` с правильным SYSTEM и параметрами:
   ```bash
   cd voxflow/ollama
   ollama create voiceflow -f Modelfile
   ```

## Подключение в приложении

VoxFlow → раздел **ИИ**:
- **Бэкенд ИИ** → «Локальный (Ollama / Qwen3)».
- **Адрес Ollama** → `http://localhost:11434` (по умолчанию).
- **Модель** → `qwen3:4b` (или `voiceflow`, если собрал из Modelfile).
- Нажми **«Проверить»** — приложение пингует Ollama и список моделей.

Включи **«Авто-стиль по приложению»**, чтобы профиль выбирался по активному окну.

## Важные детали

- **Свой системный промпт на каждый запрос.** Приложение передаёт системную инструкцию
  (`src-tauri/prompts/voiceflow_ru.txt`) при каждом вызове, поэтому обычная `qwen3:4b`
  работает без сборки `voiceflow`. Сборка `voiceflow` нужна лишь для CLI-тестов.
- **Глушение размышлений.** Для гибридной `qwen3:4b` VoxFlow добавляет `/no_think` и срезает
  блок `<think>…</think>` из ответа — задержка ниже, вывод стабильнее.
- **VERBATIM не вызывает LLM.** В редакторах кода/терминале (VS Code, Cursor, JetBrains,
  терминал) приложение НЕ зовёт модель — текст идёт «как есть» (только правила/словарь),
  чтобы не «причёсывать» код.
- **Graceful-деградация.** Если Ollama не запущена или модель не найдена — VoxFlow молча
  откатывается на локальный whisper + правила (постобработку). Диктовка не ломается.
- **Параметры (Qwen non-thinking):** temperature 0.7, top_p 0.8, top_k 20, min_p 0.0.

## Расширение динамической таблицы

Профили выбираются по активному приложению в `src-tauri/src/app_context.rs` (таблица
маршрутизации) — добавление приложения = добавление строки, переобучать ничего не нужно.
Текстовые характеристики профилей живут в системном промпте `prompts/voiceflow_ru.txt`.
