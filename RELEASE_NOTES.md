# VoxFlow v2.0.10 — мгновенная вставка в режиме auto

Patch-релиз поверх 2.0.9. Главное: в режиме языка «auto» с движком GigaAM
финальное распознавание больше не уходит на медленный Whisper large при каждой
диктовке. Задержка «отпустил клавишу → текст в поле» падает с ~2.7 с до
~0.1–0.3 с — финал теперь считает тот же быстрый GigaAM, что рисует живой
черновик в кружке.

## Auto-финал через GigaAM, Whisper — только фолбэк

- Раньше при `language=auto` кружок показывал текст мгновенно (GigaAM-preview),
  а после отпускания клавиши та же запись заново распознавалась Whisper large —
  стабильные 2.6–2.9 с на каждый финал, даже для полусекундной фразы
  (замер: `asr=2719мс … inject=18мс`).
- Теперь финал auto-маршрута сначала считает GigaAM (~0.1 с). Если результат —
  русская речь (кириллицы больше латиницы), он вставляется сразу. Латинские
  термины внутри русской фразы («right CMD», «cargo build») сохраняются.
- Whisper large подключается только когда GigaAM вернул пустой или латинский
  текст — то есть диктовка была на английском. Точность английского в auto не
  меняется, меняется лишь порядок движков.
- Если GigaAM уже отработал до Whisper, его результат переиспользуется в
  выборе «whisper vs gigaam» — второго полного прогона больше нет.
- Без установленной Whisper-модели поведение прежнее: auto отдаёт GigaAM как есть.

## Проверка

- 310 Rust-тестов пройдено (5 model/audio/platform пропущены), включая новые
  на эвристику доверия GigaAM-финалу.
- Реальная 26-секундная русская диктовка прогнана через новый путь
  (`--gigaam-selftest`): текст распознан полностью, финал-стенка 1.5 с даже в
  debug-сборке против 2.7+ с у Whisper в release.
- `check_versions.py --tag v2.0.10` — версии консистентны.
- Windows x64 и macOS ARM64 собираются и публикуются одним release workflow.
