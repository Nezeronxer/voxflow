# VoxFlow v2.0.11 — оверлей не мешает другим приложениям

Patch-релиз поверх 2.0.10. Два исправления: плавающий оверлей на macOS больше
не перехватывает курсор и клики над своими прозрачными полями, а авто-обучение
исправлений стало отключаемой опцией (по умолчанию выключено).

## Оверлей на macOS: click-through вне пилюли

- Раньше окно оверлея на macOS было всегда интерактивным, а оно крупнее видимой
  пилюли (запас под тень) — весь прозрачный прямоугольник ловил курсор: появлялась
  «ладошка», и клики не доходили до приложений под оверлеем.
- Теперь окно стартует click-through, как на Windows. Мышь включает hover-poller
  только пока курсор над hit-rect самой пилюли; курсор берётся из `cursor_position`,
  состояние ЛКМ — из `CGEventSourceButtonState`. Во время зажатой ЛКМ состояние не
  переключается — перетаскивание пилюли не рвётся.

## Авто-обучение исправлений — опция, по умолчанию выкл

- Автоматическое создание пар «распознано → правильно» из ваших правок надиктованного
  текста — эвристика, которая могла плодить неточные исправления, бьющие по нормальным
  словам. Теперь это настройка `learn_corrections` (Персонализация → Исправления),
  выключенная по умолчанию: окно обучения просто не открывается.
- Ручные пары из раздела «Исправления» работают всегда, независимо от флага, и
  по-прежнему применяются строго по границам слов.

## Проверка

- 310 Rust-тестов пройдено (5 model/audio/platform пропущены); фронт — tsc + build
  чистые.
- `check_versions.py --tag v2.0.11` — версии консистентны.
- Windows x64 и macOS ARM64 собираются и публикуются одним release workflow.
