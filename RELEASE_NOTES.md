# VoxFlow v2.0.4 — надёжное переназначение горячих клавиш

Этот patch-релиз исправляет ситуацию, когда после выбора новой клавиши
диктовка или улучшение выделенного больше не запускались.

## Горячие клавиши

- Переходы нативного режима назначения теперь выполняются строго по порядку.
  Запоздалый асинхронный `true` больше не может оставить глобальный listener
  навсегда приостановленным после выхода из настройки.
- Новая клавиша сохраняется в backend до возобновления глобального listener.
  Первое нажатие сразу после назначения больше не проверяется по старой binding.
- Повторные `keyup` во время сохранения отбрасываются, поэтому одно физическое
  назначение не создаёт несколько конкурирующих записей настроек.
- Ошибка сохранения теперь остаётся видимой в поле назначения вместо тихого
  перехода к неработающей клавише.
- macOS modifier-события используют logical flag как fallback, если системное
  переназначение клавиши или внешняя клавиатура не выставляет ожидаемый HID-state.
  Это исправляет, в частности, назначение `Right Cmd`.

## Проверка

- Добавлены regression-тесты порядка асинхронных capture-переходов и
  восстановления после ошибки, а также remapped macOS modifiers.
- Проверены TypeScript, production frontend build и полный Rust unit suite.
