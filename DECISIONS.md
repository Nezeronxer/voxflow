> Режим off-focus: решения принимаются автономно, без вопросов пользователю.

## D-000 · Кодовое имя
**VoxFlow** (оставлено из брифа). Идентификатор приложения: `com.voxflow.app`. Папка приложения: `voxflow/`, governance-доки и git-репозиторий — в корне `wispr flow/`.

## D-001 · Оболочка: Tauri 2 (React 19 + TS + Vite 7)
Скаффолд через `create-tauri-app@4.6.2`, шаблон `react-ts`, `--tauri-version 2`.
Реальные версии на момент сборки (из сгенерированных манифестов):
- `@tauri-apps/api ^2`, `@tauri-apps/cli ^2`, `tauri 2`, `tauri-build 2`

**Почему:** малые бинари, нативные хоткеи/инжект, Rust-бэкенд. Брифом задан.

## D-002 · ASR-движок: whisper.cpp как SIDECAR-бинарь (не FFI/не сборка)
**Решение:** использовать **готовый пребилт** `whisper.cpp v1.8.6` (`whisper-bin-x64.zip`), запускать `whisper-server.exe` как управляемый дочерний процесс (HTTP на `127.0.0.1`, модель грузится один раз) + `whisper-cli.exe` как one-shot fallback.
**Почему:** на машине **нет cmake и clang** → собрать whisper.cpp через `whisper-rs`/`cmake`-крейт нельзя. Пребилт убирает C++-сборку у нас и у пользователя; брифом sidecar разрешён явно. Persistent-server даёт низкую задержку для повторной диктовки (модель не перезагружается).
**Альтернативы отклонены:** `whisper-rs` (нужен cmake), `candle-whisper` (тянет тяжёлый pure-Rust стек, медленный CPU-инференс, долгая сборка), установка cmake (добавляет build-зависимость, всё равно бандлим DLL).
**Ускорение (опц.):** `whisper-cublas-12.4.0-bin-x64.zip` для NVIDIA — переключаемый профиль движка.

## D-003 · Модель по умолчанию: large-v3-turbo Q5_0
**Решение:** дефолт `ggml-large-v3-turbo-q5_0.bin` (574 МБ) — сильный русский, вдвое меньше fp16 (1.62 ГБ). Качалка: `https://huggingface.co/ggerganov/whisper.cpp/resolve/main/<file>`.
- Смоук-тест/быстрый режим: `ggml-base.bin` (148 МБ).
- Максимум качества (опц.): `ggml-large-v3-turbo.bin` (fp16) или `ggml-large-v3.bin`.
**Модель НЕ бандлится в инсталлятор** — качается при первом запуске (выбор пользователя), чтобы установщик был лёгким.

## D-004 · Захват аудио: cpal + pure-Rust ресемпл + VAD
- **cpal** — кроссплатформенный захват с устройства (нативный sample rate/формат).
- **rubato** (pure Rust) — ресемпл в **16 кГц моно f32** (whisper.cpp требует строго 16 кГц).
- **hound** — запись 16-bit PCM WAV для передачи в whisper.
- **VAD** — простой энергетический (RMS + hangover) в чистом Rust, без C-зависимостей. Для hold-to-talk VAD не критичен (запись от нажатия до отпускания); нужен для trim тишины и авто-сегментации (Flow).
**Почему pure-Rust:** избегаем `webrtc-vad`/C-сборок, минимум зависимостей.

## D-005 · Глобальный хоткей / hold-to-talk: rdev
**Решение:** **rdev** (low-level global key listener: `WH_KEYBOARD_LL` на Windows, `CGEventTap` на macOS) как основной движок хоткеев.
**Почему:** `tauri-plugin-global-shortcut` регистрирует акселераторы-комбинации и не умеет hold-to-talk на одиночном модификаторе (нет keyup для bare-модификатора). rdev даёт keydown/keyup для ЛЮБОЙ клавиши → честный hold-to-talk и toggle единым механизмом, pure-Rust.
**Дефолтная клавиша:** **Right Ctrl** (hold-to-talk), настраиваемая. Toggle — отдельная назначаемая комбинация.
**Про Fn:** на Windows Fn обычно обрабатывается прошивкой и не виден ОС (как и предупреждал бриф) → дефолт не на Fn. Если конкретная клавиша перехватывается — даём выбрать в настройках.
**Грабля:** rdev `listen` блокирует поток и не grab'ит по умолчанию → крутим на выделенном потоке, не подавляя клавишу (Right Ctrl сам по себе безвреден для чужих приложений).

## D-006 · Инжект текста: arboard (буфер) + enigo (paste/печать)
**Решение:** основной путь — **clipboard-paste**: сохранить текущий буфер → положить распознанный текст → послать **Ctrl+V** (enigo) → восстановить буфер.
Fallback — **enigo type** (посимвольно), если paste недоступен.
**Почему paste-первым:** надёжно для Юникода/кириллицы и длинного текста, быстрее посимвольной печати.
- **arboard** — кроссплатформенный буфер (save/set/restore), pure-Rust/winapi.
- **enigo** — синтетические нажатия (Ctrl+V и type-fallback), на Windows через winapi (без C-сборки).
**Off-focus:** окно настроек скрыто (работаем из трея), overlay — non-activating (`focus:false`), фокус ОС остаётся на приложении пользователя → Ctrl+V уходит в их окно. На macOS позже добавим AXUIElement для умной вставки.

## D-007 · Хранилище: rusqlite (bundled SQLite)
**Решение:** **rusqlite** с фичей `bundled` (компилит `sqlite3.c` через `cc`-крейт; cc сам находит MSVC через vswhere, cmake не нужен).
**Почему:** одна C-единица трансляции, надёжно; даёт SQL для словаря/сниппетов/истории/статистики/настроек (бриф требует SQLite).
**План B (если cc-сборка sqlite сломается):** настройки/словарь в JSON-файлах (serde) до починки. Проверяется сборкой.

## D-008 · LLM-постобработка: rule-based по умолчанию, LLM опционально
**Решение:** дефолт — **детерминированный rule-based** очиститель (паразиты, капитализация, авто-пунктуация-добивка) в Rust, полностью офлайн и быстрый. Опционально — LLM (локальный **Ollama** если запущен, либо облако Claude/OpenAI BYOK) для тона/Command Mode.
**Почему:** whisper для русского уже расставляет пунктуацию прилично; тяжёлый LLM не обязателен для рабочего среза и противоречил бы анти-оверинжинирингу. **Verbatim-режим** = полностью выключенная постобработка.

## D-009 · Структура процессов и окон
- **Трей** — основная точка (tauri tray icon), окно настроек открывается из трея, по умолчанию скрыто.
- **Overlay** — маленькое non-activating окно-индикатор записи (`focus:false`, `skipTaskbar:true`, `alwaysOnTop:true`).
- **Sidecar whisper-server** — управляемый дочерний процесс, поднимается при старте/первой диктовке, гасится при выходе.

## D-010 · Автозапуск — только с явного согласия
`tauri-plugin-autostart`, по умолчанию ВЫКЛ, включается тумблером в настройках (бриф: автозапуск только с явного согласия).

## D-011 · Загрузка моделей — системный curl, не reqwest
**Решение:** качать модели через `curl.exe` (есть в Windows 10+), прогресс — поллингом размера `.part`-файла; событие `model:progress/done/error`.
**Почему (вскрыто при сборке):** `reqwest 0.13` тянет `hyper-rustls → rustls → aws-lc-rs → aws-lc-sys`, а `aws-lc-sys` требует **cmake/nasm** (которых нет). curl использует системный SChannel — ноль C-сборок, и заодно убирает тяжёлый async-стек (reqwest+tokio+futures-util удалены).
**Альтернатива отклонена:** `reqwest` с `rustls`/`native-tls` — всё равно вытягивал aws-lc через platform-verifier.

## D-012 · rusqlite пин на 0.32 (libsqlite3-sys 0.30)
**Решение:** `rusqlite = "0.32"` (тянет `libsqlite3-sys 0.30.1`).
**Почему (вскрыто при сборке):** `rusqlite 0.40` → `libsqlite3-sys 0.38`, чей `build.rs` использует нестабильный макрос `cfg_select!` → E0658 на stable `rustc 1.93.1`. Версия 0.30 его не использует. API (`Connection`, `params!`, `query_map`, `execute_batch`) полностью совместим.

## D-013 · Запускать ТОЛЬКО продакшен-бинарь от `tauri build` (не `cargo build`)
**Грабля (вскрыто в рантайме):** голый `cargo build --release` в Tauri 2 даёт бинарь, который **остаётся в dev-режиме** и грузит фронтенд с дев-сервера `http://localhost:1420`. Без запущенного Vite WebView2 показывает страницу-ошибку Edge → выглядит как «открылся браузер». Подтверждено `netstat`: `SYN_SENT → 127.0.0.1:1420`.
**Правило:** для запуска/демо/раздачи использовать бинарь из **`tauri build`** (он же в инсталляторе) — фронтенд вшит в .exe, localhost не дёргается (проверено netstat: к 1420 обращений нет). Для разработки — `npm run tauri dev` (он сам поднимает Vite). `cargo build` использовать только для `cargo check`/линковки, НЕ для запуска GUI.

## D-014 · GPU-ускорение (cuBLAS) + persistent whisper-server
**Контекст:** у пользователя RTX 4060 Ti (16 ГБ), i5-12400F, 32 ГБ. Замеры (фраза ~9с):
- CPU fp16 large, `whisper-cli` one-shot: **~15с** (≈1.6× реального времени) — неюзабельно.
- CPU fp16 large, `whisper-server` (модель в RAM): тоже **~15с** — узкое место не загрузка, а вычисления.
- **GPU (cuBLAS) + `whisper-server`:** прогрев ~2с (разово), горячий запрос **0.36с** (≈25× realtime). ✅
**Решения:**
- Брать **cuBLAS-сборку** whisper.cpp v1.8.6 (`whisper-cublas-12.4.0-bin-x64.zip`) — она самодостаточна: внутри `cudart64_12`, `cublas64_12`, `cublasLt64_12`, `ggml-cuda`, `nvrtc` (нужен только драйвер NVIDIA). Положена в `resources/whisper-cuda/Release/` (урезана до 12 файлов).
- `paths::whisper_dir` отдаёт приоритет CUDA-сборке при наличии NVIDIA (детект: `System32\nvcuda.dll`), иначе CPU.
- **Грабля:** релизный бинарь cuBLAS скомпилён под `ARCHS=520` (sm_52), а 4060 Ti — sm_89 → ядра **JIT-компилируются из PTX через nvrtc при первом инференсе** (~2с). Поэтому `whisper-cli` one-shot медленный (JIT каждый раз), а `whisper-server` — JIT один раз → дальше быстро. ⇒ движок по умолчанию **whisper_server** + **прогрев на старте** (`engine::warmup`: поднять сервер и прогнать тишину).
- `whisper-server` управляется как дочерний процесс (порт 8771), запросы `/inference` через `curl` (multipart, `response_format=text`); гасится при выходе. Фолбэк на `whisper-cli` при сбое.
- **Инсталлятор пока бандлит только CPU-сборку** (мала, универсальна). GPU-сборка (~700 МБ, в осн. `cublasLt` 473 МБ) — локальные resources (dev-путь), для раздачи — отдельный «GPU-пак» (TODO).

## D-015 · UX-фичи (по запросу пользователя)
- **Двойное-нажатие-защёлка** (`hotkey.rs`): удержание = talk; первый release всегда немедленно останавливает запись, а второй быстрый press начинает новую запись и защёлкивает её; одиночное нажатие в защёлке = выкл. Никакого фонового отложенного `Stop` нет (`QUICK_TAP_MAX=180мс`, `DOUBLE_WINDOW=300мс`).
- **Звуки** старт/стоп: Windows `MessageBeep` (FFI, без зависимостей), гейт `play_sounds`.
- **Автозапуск:** reconcile на старте + `--autostart` флаг (при автозапуске стартуем в трее, окно не открываем) + логирование.
- **Обучение на речи** (`personalize`, по умолч. вкл): сбор пар (аудио↔текст) в `dataset/` + таблицу `samples`; адаптивный biasing — частые слова пользователя из истории/датасета в `--prompt` whisper. LoRA-дообучение — TODO (нужен Python-пайплайн).
- **Модель по умолчанию** → `ggml-large-v3-turbo.bin` (fp16, мощная); `resolve_model` при отсутствии берёт самую большую установленную.

## D-016 · «Умная система»: гейт уверенности против галлюцинаций whisper
**Жалоба:** «очень плохое распознавание». **Диагностика по РЕАЛЬНЫМ записям пользователя** (приложение сохраняет их в `dataset/`):
- Длинная чёткая речь (14с, 16с) — распознаётся **идеально**. Захват/ресемпл/уровень звука — в норме (peak 0.84, -22 dBFS, онсет не обрезан).
- Короткие/невнятные фрагменты → **галлюцинации** («Яблоковцы», «Я прокурс», «А пракос»). Это врождённая болезнь whisper на коротком/нечётком звуке, а приложение вставляло этот бред.
**Что НЕ помогло:** fp16 vs q5, beam search, `--suppress-nst` (через сервер без эффекта). Праймящий prompt и «словарь частых слов» в `initial_prompt` — **ухудшали** (prime подавлял даже хорошие записи).
**Решение (verbose_json + гейт уверенности):** запрашиваем `response_format=verbose_json`, парсим (serde_json) пословные `probability`, `no_speech_prob`, `detected_language`. Отклоняем вставку, если: mean(word prob) < 0.60, ИЛИ доля слов с prob<0.4 > 0.34, ИЛИ no_speech_prob > 0.70, ИЛИ детектирован НЕ русский с prob>0.6. При отклонении — НЕ вставляем мусор, играем звук ошибки + событие `norecog` («не расслышал»).
**Данные-подтверждение:** мусор — mean 0.57–0.605 (одно whisper детектил как ФРАНЦУЗСКИЙ 0.92); хорошая речь — mean ~0.9, ru 0.999. Гейт чисто разделяет.
**Также:** убран адаптивный word-salad prompt и дефолтный prime; biasing — только пользовательский словарь.

## D-017 · ИИ-слой: стиль-под-приложение + обучение на исправлениях (Google Gemini, BYOK)
По запросу пользователя. Собрано мульти-агентным воркфлоу (gemini.rs, app_context.rs, фронтенд).
- **Движок:** Google **Gemini** (Google AI Studio, BYOK, бесплатный лимит). Ключ хранится локально, шлётся в заголовке `x-goog-api-key` (НЕ в URL). Запросы — через `curl` (без reqwest). Модель по умолчанию `gemini-2.5-flash`. **Ключ берёт пользователь** (заводить аккаунты/брать креды я не вправе) на aistudio.google.com/apikey.
- **Стиль по приложению** (`tone_by_app`): `app_context::detect()` — активное окно (exe + заголовок) через winapi FFI → категория: `formal` (Gmail/Outlook/Word), `casual` (Telegram/WhatsApp/Instagram/Discord/VK), `ai` (ChatGPT/Claude/Gemini/Perplexity…), иначе `neutral`. Gemini `refine()` переписывает текст в стиле категории (для `neutral` LLM НЕ вызывается — без задержки/затрат).
- **Облачный ASR** (`cloud_asr`, опц.): Gemini `transcribe()` (аудио inline base64) как альтернатива локальному whisper. По умолчанию выкл (whisper приватнее/быстрее на GPU).
- **Обучение на исправлениях:** таблица `corrections`; применяются в постобработке (`apply_corrections`). **Авто-захват:** монитор буфера обмена — если пользователь скопировал отредактированную версию последней вставки (Jaccard 0.5–1.0, равная длина), учим пословные замены (распознано→правильно). Плюс ручной ввод в UI. Известные исправления также передаются Gemini в инструкции рерайта.
- **Pluggable:** `ai_backend` off|gemini; работает только при включённом ИИ и не-verbatim. Без ключа всё деградирует к локальному whisper + правилам.

## D-018 · Живой стрим текста в плашке + режим вставки Всегда/Авто/Никогда (чанковый псевдо-стрим)
По запросу пользователя (паритет с Aqua/Wispr — главная визуальная фича брифа). Собрано мульти-агентным воркфлоу `voxflow-streaming-pill` (design-lock → backend‖frontend → adversarial verify).
- **Выбор движка (сверено вебом, июнь 2026):** истинного потокового STT для русского в готовом виде НЕТ. whisper-server не имеет стримингового эндпоинта; sherpa-onnx online-моделей для русского нет (offline-only, issue #2435); Parakeet TDT v3 НЕ cache-aware (NVIDIA шлёт к FastConformer-стрим без русского) — все пути сводятся к чанкам. ⇒ **переиспользуем GPU whisper-server чанками**, без новых зависимостей/моделей (анти-оверинжиниринг).
- **Как:** во время записи фоновый поток `partial_loop` каждые ~700мс снимает накопленный буфер (`Capture::buffer_handle()` — клон Arc, НЕ сливает), ресемплит, пишет `tmp/partial.wav` (ASCII, R5), гонит `asr::transcribe_server_partial` (**без гейта** — показываем даже неуверенное) → событие `partial {text}` → плашка рисует текст серым. Замерено на GPU: партиал 0.33–0.74с (≪ каденса 700мс для нормальной речи).
- **Сериализация:** один whisper-server/порт → общий `asr_lock` (тики через `try_lock` пропускают тик, если занят; финал блокирующе). На стопе петля останавливается флагом и **джойнится** до финала. Финальный проход — с гейтом уверенности (как раньше), мусор в чужое окно не улетает.
- **Режим вставки `stream_mode` ∈ {never|auto|always}, дефолт `auto`:** плашка стримит ВСЕГДА (когда движок `whisper_server`); вставка в поле — по режиму. `never` — только финал чистовиком (как было); `always` — каждый партиал клавишами через `inject_incremental` (общий char-префикс → backspace+дозапись, корректно для кириллицы); `auto` — печатаем лишь СТАБИЛЬНЫЙ токен-префикс (совпавший в двух последних партиалах), волатильный хвост — только в плашке. Живая вставка — ТОЛЬКО клавишами (не clipboard: paste не умеет backspace).
- **Защита:** при смене активного окна (отпечаток exe+title через `app_context::detect`) живая вставка глушится до конца диктовки (`abort`) — не печатаем в чужое поле. При отклонении финала гейтом, если уже напечатали партиал, экран НЕ стираем.
- **CPU-фолбэк:** при `engine != whisper_server` петля НЕ запускается (партиалы на CPU ~15с — неюзабельно) → плашка показывает статичное «Слушаю…», вставка только финалом.
- **`inject_lock` (фикс гонки, найден adversarial-ревью):** ВСЯ эмиссия клавиш (живые вставки + финальная реконсиляция + обычная вставка) под общим мьютексом — иначе при быстром рестарте (латч/двойной тап ≤450мс) detached-поток финала прошлой диктовки и новый партиал-цикл печатали бы вперемешку, портя текст.
- **Файлы:** `engine.rs` (петля, координация, режимы), `audio.rs` (`buffer_handle`), `asr.rs` (`transcribe_server_partial` ungated + `extract_text`), `inject.rs` (`inject_incremental`), `settings.rs`+`types.ts` (`stream_mode`), `Overlay.tsx`+`styles.css` (живой текст, пилюля до 520px), `Recognition.tsx` (селектор режима), `lib.rs` (overlay-окно 560px), `main.rs`/`lib.rs` (`--stream-selftest <wav>` — headless замер каденса/латентности).

## D-023 — GigaAM-v3 как основной русский ASR + production-ready пакет (2026-06-10)

**Решение.** Русский пайплайн переведён с whisper на **GigaAM-v3 e2e RNNT int8** (SberDevices, MIT,
HF istupakov/gigaam-v3-onnx, 4 файла ~216 МБ) через крейт **ort 2.0.0-rc.12** (статическая линковка
prebuilt onnxruntime, БЕЗ cmake — единственный вариант на этой машине). e2e-чекпойнт даёт пунктуацию,
капитализацию и ITN из коробки → LLM-рефайн для «просто нормального текста» не нужен.
Замер (release, CPU): 2с фразы → 60 мс, 16с → 344 мс, против 0.36с на GPU whisper-server и 1.6–2.7с Groq.

**Сопутствующее.**
- `gigaam.rs`: фронтенд log-mel64 повторяет onnx_asr бит-в-бит (DFT-320 таблицами, bf16-округление
  окна/мел-банка), greedy transducer с кэшем dec_out. Грабля: `ort::Error<R>` не Send+Sync → адаптер `.oc()`.
- `vad.rs`: Silero VAD v6 (ресурс resources/vad, 2.3 МБ, вшит в инсталлятор). Два резидентных инстанса:
  стриминговый (петля партиалов) и финальный — общий ломал бы state при rapid-fire.
- Партиалы без GPU: сегментная схема (пауза ≥600мс или 25с закрывают сегмент → committed монотонен
  по построению; тишина не распознаётся вообще). PrefixStabilizer остался только для whisper-пути.
- Длинное аудио: финал режется по VAD-паузам на куски ≤25с (лимит pos_emb модели).
- `inject.rs`: воркер + FIFO-очередь (порядок гарантирован каналом), is_busy() глушит clipboard_monitor.
  Дефолт paste_method=clipboard (печать "type" длинного текста = старое «зависание»).
- `db.rs`: WAL + busy_timeout + авто-восстановление при malformed (у пользователя БД была битая —
  приложение не стартовало; настройки восстановлены из сырых байтов).
- Orb-оверлей по спеке Aqua Voice (снята из app.asar): окно ужимается под пилюлю (overlay_box),
  НЕ click-through, drag + память позиции (kv overlay_pos), событие level (RMS 33мс) для баров.
- Темы system/light/dark (data-theme, localStorage vf-theme до рендера).
- Дефолты: stt_provider=local, engine=gigaam, язык ru; облачные пресеты (Groq и др.) остались опцией,
  ключ пользователя сохранён. en → whisper (GPU server / cli), Parakeet отвергнут (+650МБ, слабый русский).
- Инсталлятор 0.2.0 (Inno): + resources/vad; модели GigaAM качаются при первом запуске автоматически
  (models.rs ensure_default_models, суммарный прогресс, докачка), вшивать 216 МБ в setup не стали.
