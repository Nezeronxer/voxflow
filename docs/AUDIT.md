## Архитектурные проблемы

1. **Нет абстракции STT.** Трейтов/enum нет; роутинг — каскады `if` по строковым полям в ТРЁХ местах
   (`process_utterance`, `maybe_start_partial_loop`, `start_capture_into`/`no_model_installed`);
   условие `cloud_key_ok` скопировано 3 раза. Входы движков рассинхронизированы: GigaAM ест `&[f32]`,
   whisper/cloud — путь к WAV. Добавление движка = правки в ≥6 файлах.
2. **engine.rs — god-файл 2143 строки**: оркестрация + 3 петли партиалов + финальный пайплайн +
   сегментация + стабилизатор + clipboard-обучение + промпты. Блок эмиссии partial скопирован 3 раза.
3. **Стрингли-типизация всех режимов** (engine/stt_provider/stream_mode/tone/ai_backend/...) —
   опечатка молча падает в else-ветку; дублируется в types.ts.
4. **Мультиязычность ограничена**: GigaAM захардкожен на `language=="ru"`; en/auto → только whisper
   (на CPU медленный и без партиалов); автодетекта языка нет вовсе.
5. Настройки — один JSON-blob в kv: запуск старой версии затирает новые поля.
6. Двойной дистрибутив: NSIS-конфиг в tauri.conf (рудимент D-019) + фактический Inno Setup `.iss`.
7. `trim_silence` с жёстким RMS-порогом 0.01 — тихий микрофон может срезаться до VAD;
   самописный ресемпл (moving average + linear) без тестов.
8. Гейт уверенности/dedup_repeats есть только в whisper-пути; gigaam/облако идут без анти-повторной защиты
   (RNNT практически не галлюцинирует — риск низкий, но dedup дешёвый).

### Спека Parakeet для имплементации (из протобуфа графов и исходников onnx-asr)

- Encoder: `audio_signal` f32 [B,128,T], `length` i64 [B] → `outputs` [B,1024,T/8], `encoded_lengths`.
- Decoder_joint: `encoder_outputs` [B,1024,T1], `targets` **i32** [B,U], `target_length` **i32**,
  `input_states_1/2` f32 [2,B,640] → `outputs` [B,T1,U,**8198**] (8193 vocab+blank + **5 duration-бинов**), `output_states_1/2`.
- Препроцессор: ГОТОВЫЙ ОНOX-граф `nemo128.onnx`: `waveforms` [B,N], `waveforms_lens` → `features` [B,128,T]
  (16кГц, n_fft 512, win 400, hop 160, preemph 0.97, slaney mel, log(+2^-24), норм. per-feature ddof=1).
  Писать 128-мел фронтенд руками НЕ нужно.
- TDT greedy: на шаге `token=argmax(out[:8193])`, `step=argmax(out[8193:])` (прыжок 0–4 кадра);
  token≠blank(8192) → эмит + обновить LSTM-state; step>0 → t+=step; иначе (blank или 10 эмиссий) → t+=1.
- LID-выхода НЕТ: язык определяется по скрипту выходного текста (ru/en: кириллица vs латиница).
- Скорость: RTFx int8 31.4 на 9800X3D → на i5-12400F ~10–15 → фраза 5с ≈ 0.3–0.5с. Реалтайм ок.
- 25 языков, пунктуация+капитализация из коробки. Vocab: SentencePiece `токен id`, `▁`→пробел, blank=8192.
