## 2. Мультиязычность (архитектура)

Схема из брифа подтверждена с одним уточнением: у Parakeet ONNX **нет LID-выхода** (проверено
протобуфом графов), язык определяется по скрипту его же транскрипта. Роутинг:

```
language="ru"  → GigaAM-v3 (как сейчас; Parakeet не нужен)
language="en"  → Parakeet TDT v3 (пунктуация из коробки; фолбэк whisper → ошибка «нет модели»)
language="auto"→ VAD-сегменты → Parakeet транскрибирует сегмент →
                 скрипт текста кириллический? → СЕГМЕНТ перегоняется через GigaAM (RU-качество выше)
                 иначе → текст Parakeet остаётся (EN/др.)
                 Parakeet недоступен → GigaAM (ru) / whisper (фолбэк) — как сейчас.
```

- Смешанная RU/EN диктовка работает по-сегментно (VAD-паузы режут фразы).
- Цена auto для RU-сегмента: +1 прогон Parakeet (~0.3–0.5с на фразу 5с) — приемлемо; в режиме "ru" цены нет.
- **Абстракция STTEngine**: trait `LocalStt { transcribe(&mut self, &[f32]) -> Result<String> }`
  для GigaAM и Parakeet (резиденты ort) — партиал-петля и финал становятся generic. Whisper (sidecar-процесс,
  вход WAV) и облако НЕ натягиваем на трейт — лишняя абстракция.
- Партиалы: та же сегментная схема для en/auto через Parakeet (каденс 500мс; сегмент-кап 20с).
  В auto партиалы гонит Parakeet (без двойного прогона), RU-перегон — только в финале.
- UI: язык Авто/RU/EN уже есть в Models.tsx; добавляется карточка Parakeet + бейдж языка в overlay.
- Модель Parakeet (~640МБ) НЕ качается автоматически — только по кнопке или при выборе EN/Авто (подсказка).

## 3. Качество распознавания

- RU уже на e2e RNNT (пунктуация из коробки) — без изменений; EN переходит с whisper-q5 на Parakeet e2e
  (тоже пунктуация из коробки) — главный прирост качества EN.
- 16кГц mono соблюдено; фронтенды бит-точные (GigaAM сверен ранее; Parakeet — через готовый nemo128.onnx).
- VAD: пороги 0.5/0.35, min_silence 600мс, speech_pad 300мс — уже на местах; добавить только
  отсутствие среза краёв в партиал-сегментах (PAD у границы сегмента в петле, как в финале).
- Анти-повтор dedup для всех движков (не только whisper).
- **WER до/после**: синтетический набор SAPI (RU: Irina, EN: David/Zira; 16кГц mono WAV; 8+8 фраз
  с эталонными текстами) + спот-чек на реальных записях dataset. «До» = whisper-q5 (EN) / GigaAM (RU),
  «после» = Parakeet (EN) / GigaAM (RU). Замер до начала правок и после этапа 2.

## 4. Фичи (реализуются в фазе 4, после стабилизации)

Уже есть и НЕ дублируем: словарь, сниппеты, история (read-only), autostart, трей, push-to-talk+toggle+защёлка,
оверлей с уровнем/статусом. Выбор (финализируется после стабилизации):

1. **Голосовые команды редактирования** («с новой строки», «абзац», «удали последнее предложение»,
   «отмена») — самый частый недостающий сценарий ежедневной диктовки. Распознаются на финальном тексте.
2. **История: поиск + повторная вставка/копирование** — History.tsx сейчас read-only список; поиск по
   подстроке + кнопка «копировать» делают её рабочим инструментом.
3. **Бейдж определённого языка** в оверлее и трее (RU/EN/auto-результат) — обратная связь мультиязычности.
4. **Быстрое переключение языка из трея** (Авто/RU/EN) — без открытия настроек.
5. (опц., если останется бюджет) Экспорт/импорт словаря и исправлений (JSON).

Критерии: без новых тяжёлых зависимостей; не ломает простоту; каждая доводится до конца с проверкой.
