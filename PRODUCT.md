# Product

## Register

product

## Users

macOS and Windows users who dictate into chat apps, browsers, documents, IDEs and Codex-like coding tools. They want fast voice input without accidentally sending messages, leaking keys or fighting technical settings.

## Product Purpose

VoxFlow provides free-for-personal-use speech-to-text for everyday desktop work. It should make local dictation feel predictable, private and easy to tune per application, with BYOK cloud options clearly marked as optional.

## Brand Personality

Calm, technical, direct. The interface should feel like a focused desktop utility, not a marketing page.

## Anti-references

Avoid generic AI dashboards, decorative cards, misleading "everything offline" claims when BYOK exists, and release/QA controls exposed as normal user features.

## Design Principles

- Put the current workflow first: active app, chosen profile and dictation safety are more important than implementation details.
- Make privacy state explicit: local and BYOK modes must be easy to distinguish.
- Prefer recognizable app choices over raw configuration tables.
- Keep advanced controls available, but secondary to the everyday picker.
- Use dense, quiet product UI with consistent controls and keyboard-safe behavior.

## Accessibility & Inclusion

Target readable contrast, keyboard access for settings, reduced surprise in motion and clear labels for controls that affect insertion or provider behavior.
